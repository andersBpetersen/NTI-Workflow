# i18n i NTI Workflow

## Autoritative locale-filer

Runtime bruger kun disse locale-filer i `static/i18n/`:

- `cs-CZ.json`
- `da-DK.json`
- `de-DE.json`
- `en-GB.json` (canonical source + fallback)
- `es-ES.json`
- `fi-FI.json`
- `fr-FR.json`
- `it-IT.json`
- `nl-NL.json`
- `no-NO.json`
- `pl-PL.json`
- `pt-BR.json`
- `sv-SE.json`

## Canonical source

`en-GB.json` er canonical key source. Alle øvrige locale-filer valideres mod samme key-struktur.

## Fallback-regler

Fallback-kæden i `static/i18n.js` er:

1. Valgt locale
2. Normaliseret basissprog-match
3. `en-GB`
4. Læsevenlig tekst udledt fra key-navn

I development logges manglende keys med `console.warn(...)`.

## Normaliseringsregler

Normalisering sker centralt i `normalizeLocale()` i `static/i18n.js`.

Eksempler:

- `da` -> `da-DK`
- `en` -> `en-GB`
- `en-US` -> `en-GB`
- `de` -> `de-DE`
- `pt` -> `pt-BR`
- `pt-PT` -> `pt-BR`
- `nb` -> `no-NO`
- Ukendt locale -> `en-GB`

## Fælles storage og event

- LocalStorage-key: `nti.locale`
- Eventnavn: `nti:locale-changed`
- Event payload: `{ locale: "<locale>" }`

Alle moduler (`/`, `/workflow/`, `/vault-config/`) bruger samme key og event.

## Sprogvælger

Sprogvælgeren bygges dynamisk fra `SUPPORTED_LOCALES` i `static/i18n.js`. HTML-sider indeholder ikke separate hardkodede locale-lister.

## Islandsk beslutning

Islandsk (`is.json`) er fjernet fra runtime og registry i Fase 3, fordi projektet nu konsoliderer til de 13 autoritative locale-koder ovenfor.

## Norsk beslutning

Norsk er konsolideret til én canonical locale: `no-NO`.

`nb.json` er fjernet, og `nb` normaliseres til `no-NO` i `normalizeLocale()`.

## Tilføj nyt sprog

1. Opret `<locale>.json` i `static/i18n/`.
2. Tilføj locale i `SUPPORTED_LOCALES` i `static/i18n.js`.
3. Brug samme key-struktur som `en-GB.json`.
4. Kør tests for i18n-validering.

## Validering

I18n-validering ligger i `tests/test_i18n_locales.py` og kontrollerer bl.a.:

- Registry-filer findes.
- Locale-JSON kan parses.
- Samme key-struktur som `en-GB.json`.
- Ingen tomme værdier.
- Normaliseringsregler.
- Fælles storage-key og locale-bevarelse mellem sider.
